Resumo por semana Filtro informativo usando a semana operacional quinta–quarta
Backup e exportação

Os dados ficam salvos neste navegador. Faça backup com frequência: exporte para JSON ou Excel. Também é possível importar a planilha original Indicador Semanal em formato .xlsx, puxando Produção, Reprovados e Indicador Semanal. O site também identifica a bitola automaticamente a partir do tipo do dormente.

Dados de demonstração

Carregue alguns registros de exemplo para experimentar o sistema e ver os gráficos preenchidos. Isto não afeta seus dados reais se você já tiver lançado algo (será perguntado).

Zona de risco

Apaga todos os dados salvos neste navegador. Faça um backup antes. Esta ação não pode ser desfeita.

Como funciona / próximos passos

Hoje (GitHub Pages): o site roda 100% no navegador. Cada pessoa que abre o site tem os seus próprios dados localmente. Para compartilhar, exporte o JSON e a outra pessoa importa.

Futuro (banco de dados): quando você tiver um banco, basta trocar o arquivo js/store.js para que os métodos listar / salvar / remover conversem com sua API. Todo o restante do site continua igual.

Para publicar: suba a pasta inteira para um repositório no GitHub e ative o GitHub Pages em Settings → Pages → branch main.
